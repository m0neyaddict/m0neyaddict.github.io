writing a smbios parser from scratch
what i learned building a header-only C++ library for parsing SMBIOS firmware tables on Windows.

i've been working on some stuff lately that required me to read and validate SMBIOS data. seemed simple enough at first, just grab the data and read the fields. turns out it's not that straightforward.

the first problem is just getting the data. Windows exposes it through GetSystemFirmwareTable with the RSMB signature, which internally goes through NtQuerySystemInformation. what you get back is a raw blob with an 8-byte header followed by a sequence of SMBIOS structures packed back to back.

raw parsed SMBIOS structures

the second problem is the one that catches everyone off guard. fields aren't strings, they're indexes into a string table.

once i got that working correctly the bigger surprise was how much data is actually sitting there.

processor + memory breakdown

ended up turning it into a proper header-only library. zero dependencies, drop it in and include.

source: github.com/m0neyaddict/smbios-parser
